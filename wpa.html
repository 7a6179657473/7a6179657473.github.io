WPA2 hash Gathering Project

Exploring Network Security and Wireless Authentication trends

Introduction

This project focuses on analyzing WPA2 protocols, their vulnerabilities, and the techniques used to gather relevant network data securely and ethically examine it.

Objective

  • Understand the sequence of events during WPA2 authentication and encryption.
  • Collect a wide array of hashes valid for Hashcat bruteforcing.
  • Crack and identify hashes.

Methodology

The project was conducted using ethical hacking techniques, including:

  • Network packet sniffing using software and hardware tools such as Wireshark, Pineapple MK7, and Pwnagotchi.
  • Utilizing a Pwnagotchi device for mobile hash discovery while on the move.
  • Analysis of WPA2 handshakes, distinguishing between half and full handshakes.
  • Attacking the hashes using Hashcat, gathering successful and unsuccessful attempts, specifically utilizing the rockyou wordlist and combinations of rockyou with other small wordlists.

Results

The study highlighted potential trends in end-user WPA2 configurations and provided insights into user behavior.

Data gathering involved various tools monitoring the raw traffic of WPA2 routers. The main focus was on the 2.4GHz frequency, but occasionally 5GHz was targeted.

Conclusion

By understanding the behavior and risks associated with password selection, this project aims to better understand popular trends in password security and the regions from which they are captured.

During testing, an expected trend of personally named WiFi networks resulted in a higher likelihood of weak passwords.
